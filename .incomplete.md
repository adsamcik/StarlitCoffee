## LiteRT-LM #2028 — minimal repro investigation (RESOLVED: does NOT reproduce on 0.12.0)
Context: StarlitCoffee's vision pass carries a one-multimodal-inference-per-process
guard (`visionInferenceConsumed`), inherited defensively from Mindlayer's #2028
docs ("the SECOND `Engine.close()`+new `Engine()` in a process SIGSEGVs in
`liblitertlm_jni.so`; the dispatch-delegate registry keeps dangling pointers and
the next init reads freed memory in `magic_number_utils.cc`"). To verify whether
that guard is still needed, a standalone repro was built at
`G:\Github\litertlm-2028-repro` (zip on Desktop: `litertlm-2028-repro.zip`, ~64KB).

It depends ONLY on `com.google.ai.edge.litertlm:litertlm-android:0.12.0` (the SAME
version Mindlayer uses — confirmed in Mindlayer `gradle/libs.versions.toml`), no
wrappers, runs in an isolated `:ml` process, model = `gemma-4-E2B-it.litertlm`, CPU.

**Result (2026-06-30, x86_64 Android-36 emulator): the crash does NOT reproduce.**
Tested, all clean (no `signal 11`, no tombstone, `magic_number_utils.cc` reloads fresh):
- two image inferences on one engine;
- 1× close/recreate (main process AND isolated `:ml` process);
- **5× close/recreate cycles, vision each** (disproves "the 2nd cycle SIGSEGVs");
- **5× close/recreate cycles, TEXT then VISION each** (Mindlayer's multi-pass flow).

Conclusion: the documented close/recreate trigger alone does NOT crash 0.12.0. The
real Mindlayer-stack crash (a confirmed `signal 11` tombstone seen earlier) must
depend on a condition the minimal harness doesn't exercise. Untested candidates:
(a) GPU vision backend with a thermal GPU→CPU **switch** (SwiftShader emulator
can't run GPU), (b) a bound cross-app service receiving images over **binder IPC**,
(c) **warm-slot conversation reuse** with KV-cache eviction (Mindlayer reuses
conversations; the repro makes a fresh one each call). OR it is fixed in 0.12.0.

Implication for StarlitCoffee: the `visionInferenceConsumed` one-shot guard may be
over-conservative for 0.12.0. Do NOT remove it yet — the real crash was Mindlayer-
specific and unisolated. To settle it definitively, reinstall Mindlayer
(`installDebug` in `G:\Github\Mindlayer` + StarlitCoffee), reproduce the multimodal
scan crash, and pull the tombstone (`adb shell run-as ... ` / `dumpsys dropbox` /
`/data/tombstones`) to read the actual native frame. That backtrace is the missing
evidence. The repro app + 2.4GB model are kept installed on the emulator for this.

## Why accuracy looks "only ~51%" (read before optimizing)

Two structural facts, both verified in code:

1. **The metric is pessimistic — its ceiling is ~83%, not 100%.**
   `exactAcc = exact / total`, and `total` includes correctly-ABSTAINED cells
   (fields not on the bag that the model correctly left blank). ~17% of all cells
   are correctly-absent, so a *flawless* extractor maxes out near 83%. The honest
   "got present fields exactly right" number is **recall = 63%**. See
   `app/src/sharedTest/kotlin/com/adsamcik/starlitcoffee/test/corpus/QualityReport.kt`
   (metric defs) and `BagFieldScorer.kt` (outcomes).
   - Examples: altitude exactAcc 57% but recall 100%; variety 53% vs 83%; farm 42% vs 70%.

2. **OCR is the real ceiling on messy bags — and the benchmark can't see it.**
   `LlmCorpusBenchmarkTest` scores the **text pass only**, on cached OCR
   fixtures. Same prompt across capture tiers: Q0 (studio) 71% → Q1 51% → Q2 46%
   → Q3 (night/glare) 42% — a 29pp swing driven purely by input quality. The
   model can't extract tokens OCR dropped. The fix is the **vision pass**, which
   the benchmark does not currently exercise.

Genuinely weak fields split two ways:
- Free-text, hard to exact-match, low-stakes (user edits anyway): `name`,
  `roaster`, `tastingNotes`.
- Real model confusion, worth fixing: `roastLevel` (~10/28 wrong), `process`
  (bean-form/decaf leak — e.g. `BOHNEN·EN GRAINS` mis-mapped). Both are
  vision-addressable (marks, layout).

Also: hallucination rose 6%→20% (translate-first abstains less); single runs at
temp 0.1 wobble ±5pp.

## How to run the benchmark (do NOT use connectedAndroidTest)

`connectedAndroidTest` uninstalls the app between runs and wipes the OCR
fixtures. Use the dedicated task:

```powershell
.\gradlew.bat scanBenchmark                # capture OCR fixtures, then benchmark
.\gradlew.bat scanBenchmark -PskipCapture  # reuse fixtures (prompt-only iteration)
```

Report lands in `app\build\reports\scan-benchmark\llm-fixture-quality-report.{txt,json}`.
Full details: `.github/instructions/benchmark.instructions.md`.

Notes:
- Debug app id: `com.adsamcik.starlitcoffee.debug` (test `…debug.test`).
- On-device inference is slow (~30 min/run, ~1–2 LLM calls/bag).
- The `scanBenchmark` task auto-enables the Mindlayer debug auto-accept receiver
  so inference runs unattended.
- Use the `android-emulator-mcp` server for any on-device interaction (claim the
  device first).
